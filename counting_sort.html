Counting Sort

Θ(n+k) temporale Θ(n+k) spaziale Stabile · Non comparativo
§ Teoria
Il Counting Sort assume che gli elementi da ordinare siano interi in un intervallo noto [0, k]. Non esegue confronti — sfrutta i valori stessi come indici.
Fase 1 — Conteggio: conta quante volte compare ogni valore in A, accumulando in C.
C[i] = |{j : A[j] = i}|
Fase 2 — Prefissi cumulativi: C[i] diventa il numero di elementi ≤ i. Questo determina la posizione finale di ogni elemento.
C[i] = C[i] + C[i−1] per i = 1..k
Fase 3 — Posizionamento: scorre A da destra a sinistra (per stabilità), inserisce A[j] in B[C[A[j]]], poi decrementa C[A[j]].
B[ C[A[j]] ] = A[j]; C[A[j]] --
La scansione da destra garantisce che elementi con lo stesso valore mantengano l'ordine relativo originale → algoritmo stabile.

§ Pseudocodice · COUNTING-SORT(A, n, k)
COUNTING-SORT(A, n, k): B[1..n] = array output // risultato C[0..k] = 0 // contatori // ── FASE 1: conta occorrenze ── for j = 1 to n: C[A[j]] = C[A[j]] + 1 // ── FASE 2: prefissi cumulativi ── for i = 1 to k: C[i] = C[i] + C[i−1] // ── FASE 3: posiziona (destra→sinistra) ── for j = n downto 1: B[ C[A[j]] ] = A[j] C[A[j]] = C[A[j]] − 1 return B
① Init
C = 0
② Conta
C[A[j]]++
③ Prefissi
C[i]+=C[i-1]
④ Output
B[C[A[j]]]=A[j]
✓ Done
