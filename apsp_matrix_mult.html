Matrix Multiplication

SLOW-APSP · Θ(n⁴)
Teoria

L'algoritmo sfrutta un'analogia con la moltiplicazione di matrici. Si definisce L(m)[i][j] come il peso del cammino minimo da i a j usando al più m archi.

Caso base — matrice dei pesi diretti:

L⁽¹⁾[i][j] = W[i][j]

Passo induttivo — operatore EXTEND:

L⁽ᵐ⁾[i][j] = min1≤k≤n { L⁽ᵐ⁻¹⁾[i][k] + W[k][j] }

Dopo n−1 iterazioni si ottengono tutti i cammini minimi. La versione fast usa il repeated squaring: L⁽²ᵐ⁾ = L⁽ᵐ⁾ ⊗ L⁽ᵐ⁾, riducendo le iterazioni da n−1 a ⌈log₂(n−1)⌉.

Pseudocodice · SLOW-APSP
SLOW-APSP(W): n = W.rows L⁽¹⁾ = W // inizializzazione for m = 2 to n−1: L⁽ᵐ⁾ = EXTEND(L⁽ᵐ⁻¹⁾, W) return L⁽ⁿ⁻¹⁾ EXTEND(L, W): L' = matrice n×n inizializzata a ∞ for i = 1 to n: for j = 1 to n: for k = 1 to n: if L'[i][j] > L[i][k] + W[k][j]: L'[i][j] = L[i][k] + W[k][j] Π[i][j] = Π[i][k] // aggiorna predecessore return L'
